# 数据大屏管理系统文档

## 1. 项目概述

###  - 架构设计1.1 项目背景

随着企业数字化转型的深入，数据可视化已成为企业决策支持的核心工具。数据大屏作为展示关键业务指标（KPI）的重要载体，能够帮助企业管理者快速获取业务全貌，及时发现问题并做出决策。

本项目旨在开发一套通用的数据大屏管理平台，支持：
- 多行业、多套模板的前端展示
- 灵活的后台配置管理
- 多种数据源接入（关系型数据库、NoSQL、API接口、JSON文件）
- 模板的导入导出功能
- 可视化编辑界面

### 1.2 项目目标

1. **模板管理**：提供多套行业模板（金融、零售、制造、物流等），支持模板的创建、编辑、复制、导入、导出
2. **数据源管理**：统一管理各类数据源，支持MySQL、PostgreSQL、MongoDB、REST API、WebSocket、JSON文件等
3. **可视化编辑**：通过拖拽方式编辑大屏组件，配置数据绑定，实现所见即所得
4. **实时渲染**：支持数据的实时刷新和动态展示
5. **权限管理**：多用户协作，角色权限控制

### 1.3 项目范围

- **前端**：React + TypeScript 技术栈，实现大屏展示和后台管理界面
- **后端**：Node.js + NestJS 框架，提供API服务
- **数据库**：MySQL 8.0（主数据库）+ 可配置外部数据源
- **部署**：支持Docker容器化部署

### 2.2 架构分层说明

#### 2.2.1 客户端层

**大屏展示端（Dashboard View）**
- 使用React框架开发
- 支持1920x1080、3840x2160等标准分辨率
- 响应式布局适配不同屏幕
- 组件化开发，支持拖拽布局
- 数据实时刷新（WebSocket/轮询）
- 丰富的图表组件（ECharts）

**后台管理端（Admin Panel）**
- 使用React + Ant Design组件库
- 左侧导航菜单
- 右侧内容区域
- 表单配置、数据表格、树形控件等
- 可视化拖拽编辑器

#### 2.2.2 网关层

- **Nginx**：静态资源服务、负载均衡、反向代理
- **CORS**：跨域资源共享配置
- **认证中间件**：JWT token验证
- **限流防护**：防止恶意请求

#### 2.2.3 应用层

采用NestJS框架，模块化架构：

| 模块名称 | 功能描述 |
|---------|---------|
| TemplateModule | 模板管理（增删改查、导入导出） |
| DataSourceModule | 数据源配置（连接池、查询测试） |
| ComponentModule | 组件配置（组件库、属性编辑） |
| ApiConfigModule | 接口配置（API定义、参数映射） |
| UserModule | 用户管理（注册、登录、信息） |
| AuthModule | 权限认证（JWT、角色、菜单） |
| DataFetchModule | 数据获取（统一数据服务） |

#### 2.2.4 数据层

- **MySQL 8.0**：存储系统配置、用户信息、模板元数据
- **Redis**：缓存会话、热点数据、实时数据
- **文件系统**：存储模板JSON文件、导出文件
- **外部数据源**：动态连接各类数据库和API

- **连接池**：数据库、Redis连接复用
- **缓存**：热点数据Redis缓存
- **查询优化**：SQL预编译、索引优化
- **异步处理**：大数据量后台处理

### 6.3 实时数据优化

- **WebSocket**：长连接实时推送
- **数据聚合**：减少请求次数
- **增量更新**：只传输变化数据
